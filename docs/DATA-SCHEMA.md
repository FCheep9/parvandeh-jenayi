# Case data schema

Every case is a self-contained folder under `data/cases/<id>/`. The engine never needs
editing to add a case — only data. Register the case in `data/registry.json`.

```
data/cases/<id>/
  case.json        # meta + acts/beats (the spine)
  people.json      # interviewable characters + branching dialogue
  locations.json   # searchable scenes + hotspots
  evidence.json    # examinable items + forensic notes
  clues.json       # deduction-board clue cards, links, conclusions
  media.json       # portrait + photo URLs (generate with tools/fetch-media.mjs)
```

IDs are strings, unique within their file. Anything referenced (an `unlock`, a `clue`, a
`require.target`) must exist, or that step can't be satisfied.

## registry.json
```json
{ "firstCase": "the-last-clean-take",
  "cases": [ { "id": "...", "title": "...", "logline": "...", "playable": true } ] }
```

## case.json
```jsonc
{
  "meta": {
    "title": "string (HTML allowed)",
    "act_label": "Act 1 — …",
    "clock": "HTML string shown in the header (optional)",
    "logline": "string",
    "protagonist_note": "string (optional)",
    "coldOpen": [ { "text": "...", "sub": "...(optional)", "ms": 4200 } ]   // cinematic slides
  },
  "acts": [ { "id": "act1", "title": "...", "beats": [ Beat, … ] } ]
}
```

### Beat
```jsonc
{
  "id": "b1-arrival",                 // unique within the act
  "title": "string (optional)",
  "act_label": "kicker shown above the title (optional)",
  "image": "media slot id (optional)",
  "speaker": "person id (optional) — renders body as that character's line",
  "body": "string with \\n\\n paragraph breaks, or an array of strings",
  "callout": "HTML tutorial tip (optional)",
  "unlock": { "evidence": [], "people": [], "locations": [], "clues": [] },  // applied once
  "collect": ["clueId", …],          // auto-add clues to the board (optional)
  "advise": ["advisor/partner note", …],  // added to the Notebook (optional)
  "require": Requirement,            // gates the Continue button (optional)
  "continueLabel": "Continue ▸",     // optional
  "goto": "beatId",                  // optional non-linear jump
  "lead_only": true,                 // decision must be committed by the Lead (with choices)
  "choices": [ Choice, … ]           // presence makes this a decision beat (no Continue)
}
```

**Requirement.type** — `open-tab` (target: tab name) · `examine-evidence` (target id, or n) ·
`interview` (target person id) · `search` (target `"locId:hotspotId"`, or n) ·
`collect-clue` (target clue id) · `conclusion` (target conclusion id).
On **Casual** difficulty, requirements never hard-block (soft gates).

**Choice** — `{ id, label, detail?, setFlag?, flagValue?, unlock?, advise?, goto? }`.

## people.json
```jsonc
{ "personId": {
  "id", "name", "role", "intro": "first line",
  "dialogue": [ {
    "id", "topic": "button label", "line": "answer (HTML)",
    "press": true,                         // highlighted on Casual ("⚡")
    "requires": { "evidence"?, "clue"?, "flag"? },   // gate this topic
    "reveals": { "evidence":[], "people":[], "locations":[], "clue"?, "clues":[], "flag"? }
  } ] } }
```

## locations.json
```jsonc
{ "locId": {
  "id", "name", "slot": "media id", "summary", "description",
  "hotspots": [ { "id", "label", "text": "observation (HTML)",
                  "clue"?: "clueId", "unlock"?: { … } } ] } }
```

## evidence.json
```jsonc
{ "evId": {
  "id", "name", "slot": "media id (\"\" → placeholder)", "glyph": "emoji for placeholder",
  "summary", "description", "notes": ["HTML", …],
  "clue"?: "clueId",          // enables "+ Add to board"
  "tool"?: "waveform",        // opens a specialized tool
  "unlocks"?: { … }           // examining reveals more (once)
} }
```

## clues.json
```jsonc
{
  "clues": { "id": { "id", "title", "src", "icon" } },
  "links": [ { "pair": ["clueA", "clueB"], "conclusion": "concId" } ],
  "conclusions": { "id": { "id", "title", "text",
      "unlocks"?: {…}, "collectClue"?: "id", "setFlag"?: "name" } }
}
```

## media.json
Generated by `node tools/fetch-media.mjs` (keyless Openverse photos + randomuser portraits),
but hand-editable:
```jsonc
{ "people": { "personId": { "portrait": "https://…" } },
  "slots":  { "slotId":  { "url": "https://… or null", "credit": { … } | null } } }
```
Missing/blocked images fall back to a styled placeholder at runtime, so the game never breaks.

## Testing
`node tests/smoke.mjs` boots the app in jsdom and plays Act 1 end-to-end. Extend it when you
add a case.
